HTML

정의

웹 페이지를 위한 지배적인 마크업 언어. HTML은 제목, 단락, 목록 등과 같은 본문을 위한 구조적 의미를 나타내는 것뿐만 아니라 링크, 인용과 그 밖의 항목으로 구조적 문서를 만들 수 있는 방법을 제공한다. 그리고 이미지와 객체를 내장하고 대화형 양식을 생성하는 데 사용될 수 있다. HTML은 웹 페이지 콘텐츠 안의 꺾쇠 괄호에 둘러싸인 "태그"로 되어있는 HTML 요소 형태로 작성한다. HTML은 웹 브라우저와 같은 HTML 처리 장치의 행동에 영향을 주는 자바스크립트와 본문과 그 밖의 항목의 외관과 배치를 정의하는 CSS 같은 스크립트를 포함하거나 불러올 수 있다. HTML과 CSS 표준의 공동 책임자인 W3C는 명확하고 표상적인 마크업을 위하여 CSS의 사용을 권장한다.

Hypertext Markup Language (HTML) is the standard markup language for creating web pages and web applications.

특징

-html의 장점은 다른 언어에 비해 쉬우면서 중요하다는 점이다.
-html소스를 작성하기 위해서는 소스의 결과가 보여지는 웹브라우저와 고급언어로 소스를 작성할 수 있는 프로그램, 즉 에디터 프로그램이다.
-태그는 부모태그와 자식태그로 나뉘어진다. ex). ul태그와 li태그

태그종류

  • a: hyperlink tag
    ex). a href(hyper + reference)="link"
    / 속성은 href와 target이 있다. 새탭으로 열고싶을 경우 target="_blank"추가
    / 해당 링크설명시 title="name"
  • head: 본문을 설명하는 tag
  • body: 본문 tag
  • html: 최고위층 tag(head와 body태그를 묶음)
  • title: webpage title tag(사용자에게 웹 페이지 이름을 명시적으로 보여줌.)
  • meta: 해당 언어로 파일을 여는 tag
    ex). meta charset(charater set) = "utf-8"
  • br: new line tag
  • p: paragraph tag
  • h1~h6: heading tag(글씨가 점점 작아짐)
  • strong: bold tag
  • u: underline tag
  • img: image tag(width로 image 크기설정 가능)
    ex).img src="link"
  • li: list tag
  • ul: unordered list tag
    / li태그의 부모태그
  • ol: ordered list tag
  • div: block element tag
  • span: inline element tag
  • !--: ignored tag

CSS 관련태그

  • id: HTML 태그 안에 작성하여 해당 id명. CSS로 id명 작성 시 앞에 #을 붙인다.
  • class: HTML 태그 안에 작성하여 해당 클래스명으로 그룹을 묶을 수 있다. CSS로 class명 작성 시 앞에 .을 붙인다.
  • (추가적으로 태그 안에 작성하는 속성에는 여러 value를 할당할 수 있고, 이는 띄어쓰기로 구분된다. 그래서 ex). class="saw active"라고 하면 해당 태그는 saw와 active 클래스 모두에 속해있다. 그리고 작성순서에 따라 영향을 받기 때문에 해당 태그는 saw 클래스가 먼저 작성되면 active, active 클래스가 먼저 작성되면 saw 클래스의 영향을 받는다.)
